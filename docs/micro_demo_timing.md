# Micro Demo Timing — 10‑Second Chunks

Overview
- Break the live demo into modular 10‑second chunks so you can stop or condense at any boundary. Each chunk includes: what to show, an exact 1‑line speaker line, and cues (when to emphasize, when to pause).

Core 2‑minute flow (12 × 10s)

1) 0:00–0:10 — Hook
- Show: Title slide + one bold stat (loss or dollars).  
- Say: "Utilities lose X% in unnoticed losses — that's recoverable revenue."  
- Cue: Emphasize "recoverable revenue" (strong voice), pause 0.6s.

2) 0:10–0:20 — One‑line solution
- Show: Tagline slide with `Spot. Seal. Recover.`  
- Say: "We spot anomalies, seal evidence, and recover revenue."  
- Cue: Snap fingers / slight gesture; emphasize the three verbs.

3) 0:20–0:30 — Show the Action Queue
- Show: Live UI (Action Queue) or screenshot.  
- Say: "This is the Action Queue — high‑confidence cases at the top."  
- Cue: Hover to highlight top item; pause 0.5s for judges to read.

4) 0:30–0:40 — Trigger an alert (demo moment)
- Show: Terminal/button click to run demo trigger.  
- Say: "I'll trigger a realistic alert — watch the queue update."  
- Cue: Count quietly "3…2…1", click, then emphasize "watch".

5) 0:40–0:50 — Alert appears
- Show: Queue updating with new item; highlight message.  
- Say: "There — a prioritized alert appeared with confidence and evidence."  
- Cue: Point to confidence percent; pause 0.4s.

6) 0:50–1:00 — Open alert & evidence
- Show: Open alert, display photo/hash and provenance.  
- Say: "Photo + signed hash — auditable proof for enforcement."  
- Cue: Emphasize "auditable"; hold 1s for effect.

7) 1:00–1:10 — Dispatch to inspector
- Show: Assign or simulate mobile inspector view.  
- Say: "We dispatch only high‑ROI cases — inspector gets a one‑tap job."  
- Cue: Show mobile mock, tap 'Confirm'. Pause 0.5s.

8) 1:10–1:20 — Inspector confirms outcome
- Show: Inspector photo/signature and resolution state.  
- Say: "Inspector confirms and captures evidence — closure and value recorded."  
- Cue: Emphasize "value recorded"; brief pause.

9) 1:20–1:30 — Feedback loop
- Show: Retraining note or model version change.  
- Say: "Inspector feedback feeds retraining — precision improves quickly."  
- Cue: Emphasize "precision improves"; quick visual arrow showing improvement.

10) 1:30–1:40 — ROI snapshot
- Show: Estimated recovery, cost‑per‑inspection, and payback.  
- Say: "A small pilot pays back in months — here are typical numbers."  
- Cue: Point to the recovered revenue figure; pause 1s for impact.

11) 1:40–1:50 — Scaling path
- Show: Simple architecture / scale bullets (pilot → regional → national).  
- Say: "We scale by adding stateless workers and regional storage — low marginal cost."  
- Cue: Emphasize "low marginal cost"; one short nod.

12) 1:50–2:00 — Close & ask
- Show: Tagline + CTA slide.  
- Say: "Spot. Seal. Recover. Run a 3‑month, 5k‑meter pilot with us."  
- Cue: Deliver the three words as a rhythmic close, pause 1.5s, then the ask with eye contact.

Optional extended chunks (3‑minute demo)
- 2:00–2:10 — Deep dive on model explainability (show SHAP or features).  
- 2:10–2:20 — Walk through data quality checks and edge buffering.  
- 2:20–2:30 — Show inspector routing and expected trips saved per week.  

Presenter cues & best practices
- Pause lengths: use 0.4–1.5s pauses after important claims; pause longer after ROI figures.  
- Emphasis rules: bold the single word you want the room to remember (e.g., "revenue", "auditable", "ROI").  
- Visual timing: perform clicks or animations on counts (e.g., 3‑2‑1) so judges anticipate the moment.  
- Fallback chops: if cut to 30s → deliver chunks 1–3 (hook, solution, Action Queue). If cut to 60s → deliver chunks 1–6. If cut to 90s → deliver chunks 1–9.

Technical cues (runner)
- Start demo script 10s before chunk 4 so the alert arrives on cue.  
- If the live alert fails, switch to preloaded screenshot and narrate the same steps; call out "fallback demo" quickly.

Short script mapping (30s / 60s / 120s)
- 30s: Hook (0–10s), Solution (10–20s), Quick UI + one trigger (20–30s) — close with Tagline in last 2s.  
- 60s: Chunks 1–6 exactly as written.  
- 120s: Full core flow (chunks 1–12).

Practice tip
- Rehearse with a stopwatch and run the trigger script at least 5 times to be confident with timing and surprises.
